## Summary

RadCN has strong Combobox primitive coverage. `radcn/combobox` owns searchable
listbox behavior, empty state, grouped options, selected indicators, trigger
open/close behavior, keyboard roving, disabled options, clearable state,
multiple-selection chips, popper side/align placement, hidden input submission,
reset behavior, public hooks, and custom styling hooks. The candidate fixture
and Playwright tests prove those primitives directly.

The upstream examples are composition examples rather than only primitive
examples. They combine combobox-like searchable command lists with Button,
Popover, Dropdown Menu submenus, Drawer, icons, responsive branching, and
React-owned local state. RadCN already has most constituent primitives, but the
four named upstream examples are not yet represented as named docs examples,
candidate fixture routes, or Playwright assertions. Combobox example parity is
therefore partial and needs one implementation-depth experiment.

`combobox-form` exists as an upstream public JSON payload, but the active
parity inventory's `combobox` cluster lists exactly `combobox-demo`,
`combobox-dropdown-menu`, `combobox-popover`, and `combobox-responsive`.
`combobox-form` depends on `form`, React Hook Form, Zod, Sonner, and
form-specific validation behavior, so it should remain adjacent evidence for
Form and Combobox composition rather than a fifth row in this cluster unless a
future inventory explicitly reclassifies it.

## Mapping Decisions

| Upstream concept | RadCN mapping |
| --- | --- |
| React `useState` for `open` and selected value | Map to RadCN default/open data attributes, hidden inputs, route/server state, or small dependency-free browser enhancement where a demo needs client-side label persistence. Do not add React. |
| `onSelect` callbacks | Map to package-owned selection behavior for `radcn/combobox`; map composed label updates to app-owned browser enhancement or server-owned submitted state. |
| `Button asChild` | Map to explicit RadCN trigger components (`ComboboxTrigger`, `PopoverTrigger`, `DropdownMenuTrigger`, `DrawerTrigger`) and app-authored button classes. |
| Upstream `Command` inside examples | RadCN may use `radcn/combobox` directly for combobox behavior or compose `radcn/command` where the example is command-palette-specific. Do not depend on upstream command source. |
| Upstream `Popover` | Map to `radcn/popover` or direct `ComboboxContent` side/align placement depending on the example. |
| Upstream `DropdownMenu` | Map to `radcn/dropdown-menu`; the searchable label submenu remains a composition of menu and searchable-list primitives, not a Combobox package dependency on Dropdown Menu. |
| Upstream `Drawer` | Map to `radcn/drawer` for the responsive mobile branch. |
| `useMediaQuery` | Prefer CSS breakpoints for branch visibility, or a dependency-free browser module if state must coordinate branches. Do not add a React hook. |
| `lucide-react` icons | Icons such as `Check`, `ChevronsUpDown`, and `MoreHorizontal` are app-owned presentation. Use existing icon strategy, inline SVG, text glyphs, or `lucide-static` in docs; do not add `lucide-react` to `radcn/combobox`. |
| Tailwind utility classes | Map to RadCN classes, `class`, `style`, CSS variables, and app-authored CSS. Tailwind is not a RadCN package dependency. |
| `className` | Map to Remix UI `class`. |
| `data-slot` | Map to public `data-radcn-combobox-*`, `data-radcn-dropdown-menu-*`, `data-radcn-popover-*`, and `data-radcn-drawer-*` hooks plus stable classes. |
| Vendor source | Use vendor files only as read-only evidence. RadCN package code, docs, fixtures, tests, and pnpm dependency resolution must not depend on `vendor/`. |
| `combobox-form` | Treat as adjacent Form/Combobox composition evidence. Its React Hook Form, Zod, Sonner, and form validation mechanics are outside this four-example combobox cluster. |

## Recommendation

Combobox example parity is partial. The next experiment should implement
Combobox example parity depth by adding named docs examples, candidate fixture
routes, and Playwright coverage for `combobox-demo`,
`combobox-dropdown-menu`, `combobox-popover`, and `combobox-responsive`.

The implementation should preserve the current `radcn/combobox` package API
unless direct evidence shows a blocker. It should compose existing RadCN
Button, Combobox, Command, Dropdown Menu, Popover, and Drawer primitives,
record app-owned label/state behavior explicitly, and keep React,
`lucide-react`, Tailwind, `useMediaQuery`, and vendor source out of RadCN
package dependencies.
