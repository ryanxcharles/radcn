## Summary

RadCN already has the Badge primitive variants required by the upstream Badge
examples: `default`, `secondary`, `destructive`, and `outline`, plus RadCN
extras `ghost` and `link`. The package exposes public `data-radcn-badge` and
`data-variant` hooks, renders anchors when `href` is supplied, accepts custom
classes/styles, and allows arbitrary children, so icon and numeric content are
mechanically possible without changing the package API.

The Badge example cluster is still `Partial`, not resolved, because RadCN lacks
named docs/fixture/Playwright evidence for the four upstream Badge example ids.
The richer upstream `badge-demo` also includes app-owned icon presentation,
custom color classes, and compact numeric pill/count badges that are not
explicitly documented or tested today.

## Mapping Decisions

| Upstream concept | RadCN mapping |
| --- | --- |
| `className` | Map to RadCN `class` prop. |
| Tailwind utility classes | Map to RadCN classes, `style`, CSS variables, and app-authored CSS. Tailwind is not a RadCN package dependency. |
| `lucide-react` icon children | Map to app-owned icon presentation or simple inline content. Do not add `lucide-react` as a Badge dependency. |
| `data-slot="badge"` | Map to public `data-radcn-badge`, `data-variant`, and `radcn-badge` classes. |
| React prop spreading | Map to explicit Remix UI props: `variant`, `href`, `class`, `style`, and children. |
| Link behavior | RadCN Badge owns explicit `href` anchor rendering. Upstream Badge examples do not require links, but existing fixture coverage proves the RadCN extension. |
| Custom pill dimensions | App-owned styling through `class`, `style`, or CSS variables; Badge should not add count-specific props. |
| Numeric/count content | Plain Badge children. Formatting and count truncation remain app-owned. |
| Vendor source | Use vendor files only as read-only evidence. RadCN package code, docs, fixtures, tests, and pnpm dependency resolution must not depend on `vendor/`. |

## Recommendation

The next experiment should implement Badge example parity depth. It should keep
the package API unchanged unless a direct blocker appears, add named docs and
fixture evidence for `badge-demo`, `badge-destructive`, `badge-outline`, and
`badge-secondary`, and prove variant labels, icon composition, custom class or
style overrides, compact numeric/count badges, public hooks, and no
`lucide-react`/Tailwind/React/vendor dependency with Playwright.
